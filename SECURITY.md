## 报告漏洞

我们非常重视项目的安全性。如果您发现了安全漏洞，请通过以下方式报告：

### 报告方式

**请勿在公开的 Issue 中报告安全漏洞。**

请通过以下方式之一报告：

1. **GitHub Security Advisories**（推荐）
   - 访问仓库的 Security 标签页
   - 点击 "Report a vulnerability"
   - 填写安全漏洞报告

2. **邮件联系**
   - 发送邮件到项目维护者（如果提供了邮箱）

### 报告内容

请在报告中包含以下信息：

- 漏洞的详细描述
- 复现步骤
- 潜在影响
- 建议的修复方案（如果有）
- 您的联系信息（可选）

### 响应时间

我们承诺：

- **24 小时内**：确认收到报告
- **7 天内**：评估漏洞严重性
- **30 天内**：发布修复或提供解决方案

### 漏洞处理流程

1. **确认**: 我们会在 24 小时内确认收到报告
2. **评估**: 评估漏洞的严重性和影响范围
3. **修复**: 开发修复方案
4. **测试**: 测试修复方案
5. **发布**: 发布安全更新
6. **披露**: 在修复后公开披露（如果适用）

### 奖励

目前项目不提供漏洞奖励计划，但我们非常感激您的贡献！

## 安全最佳实践

### 对于用户

- 定期更新到最新版本
- 只从官方渠道下载
- 检查文件完整性
- 不要运行来源不明的插件

### 对于开发者

- 遵循安全编码实践
- 及时更新依赖项
- 进行安全代码审查
- 使用安全的开发工具
